7. 
| Pattern                | Meaning                                    |
| ---------------------- | ------------------------------------------ |
| `/Logged in as/`       | Contains text anywhere                     |
| `/^Logged in as/`      | Must start with text                       |
| `/Logged in as$/`      | Must end with text                         |
| `/^Logged in as .*$/`  | Must start with + anything after           |
| `/^.*Logged in as.*$/` | Anything before + contain + anything after |


8. | is called the alternation operator.
It matches either the pattern on the left OR the pattern on the right.
So:
const errorText = await page.getByText(/incorrect|Invalid/i).innerText();

10. Spread vs Destructuring vs rest 

| Operator                    | Meaning        | Real Use                  |
| --------------------------- | -------------- | ------------------------- |
| `...` inside object literal | Spread         | Copy + override user data |
| `{ key }`                   | Destructuring  | Extract login credentials |
| `{ key, ...rest }`          | Rest           | Separate remaining fields |
| `function(...args)`         | Rest parameter | Collect multiple products |

STEP 3: Running From Cloud (Two Ways)
You have 2 industry ways:
✅ OPTION 2 — Run Automatically Using GitHub Actions (Best Practice)
This is industry standard 🔥
Instead of manually creating VM, we let GitHub run tests automatically.
🔹 What is GitHub Actions?
GitHub Actions is CI/CD system inside GitHub.
It:
Detects code push
Creates temporary Linux machine
Runs your commands
Deletes machine after job
Fully cloud ☁️

STEP 4: Create GitHub Actions Workflow
Inside your repo, create folder:
.github/workflows
Inside that create file:
playwright.yml
🔹 Add This Content

name: Playwright Docker Tests

on:
  push:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Build Docker Image
        run: docker build -t playwright-tests .

      - name: Run Tests in Docker
        run: docker run --rm playwright-tests

Push this file.

🔥 What Happens Internally Now?

When you push code:
GitHub detects push
Creates temporary Ubuntu machine
Installs Docker (already pre-installed)
Clones your repo
Builds Docker image
Runs Playwright tests
Shows logs in GitHub UI
Destroys machine
No server maintenance required.
